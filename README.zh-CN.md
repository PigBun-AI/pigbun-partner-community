# PigBun Partner

[English](./README.md)

## 一个运行在本地的 AI 合伙人，帮你扛住那些不想一直盯着的事。

大多数 AI 工具都在等你下一句话。你不说，它就不动。

PigBun 想做的是另一种关系：你把一件要持续推进的事讲给 AI 合伙人听，合伙人把它收成一条业务线，然后系统持续往前推。需要你判断、授权、给账号、给文件、拍板的时候，它再回来找你。

PigBun 的核心不是“又一个聊天框”，而是：

> 你不应该为了每一步都坐在电脑前。
> 你定方向，PigBun 帮你把事情继续推着走。

这个公开仓库是 PigBun 的早期用户入口：

- 安装说明
- Bug 反馈
- 产品建议
- 路线图和版本记录
- 社区讨论

PigBun 的核心运行时代码目前还没有开源。这个仓库是公开入口，不是完整源码仓库。

## PigBun 怎么工作

主流程很简单：

1. 你和 AI 合伙人聊天。
2. 当一件事聊清楚了，合伙人会把它接下来。
3. PigBun 为这件事创建一条业务线。
4. 业务线会保留上下文、使用本地工具，并从主聊天中独立推进。
5. 需要你做决定、给账号、给文件、给密码、做授权时，PigBun 会主动回来找你。

这就是 PigBun 和普通 AI 助手的区别：它不是“一问一答”，而是“可以托付”。

## 现在可以用它做什么

PigBun 仍处于 alpha 阶段，但已经可以用于真实工作：

- 把一个产品想法变成一条持续推进的业务线。
- 让合伙人把事情拆成调研、产品、开发、部署和后续跟进。
- 让项目上下文跨对话保留下来，而不是每次重新讲一遍。
- 使用 Claude Code、Codex 等本地执行器。
- 在浏览器工作台里查看进展、阻塞、工具动作和待决策事项。

如果你是独立开发者、创业者、运营者，或者正在把一个项目从想法推进到上线，PigBun 会更适合你。

## 安装

PigBun 当前在本地运行，并通过浏览器打开工作台。

```bash
curl -fsSL https://pigbunai.com/install.sh | bash
```

安装完成后，可以用下面的命令启动：

```bash
pigbun
```

安装脚本会检查本地环境、下载最新 alpha 运行时、安装 `pigbun` 命令，并在本地打开 PigBun。

## PigBun 背后的模型仍然非常重要

PigBun 不只是一个模型。它会在模型外面加一层协作系统：业务线、记忆、工具上下文、活动记录、恢复路径和浏览器工作台。技术上，这一层经常被叫做 Harness。

这层系统会让 AI 的工作更可见、更有组织，也更容易恢复。但它不会把任何模型都变成稳定可靠的合伙人。

模型仍然要负责理解目标、判断什么时候拆分业务线、正确使用工具、从错误中恢复。如果模型本身弱或者不稳定，PigBun 的运行稳定性和交付质量都会被影响。

所以，早期严肃使用时，强烈建议选择能力更强、更稳定的基座模型。

当前推荐：

- Claude Sonnet / Claude Opus
- GPT-5.4 / GPT-5.5
- Kimi 2.6

未来 PigBun 会支持 Kimi CLI 作为新的接入方式。

## 当前状态

PigBun 仍处于 alpha 阶段。

它现在可以用，但还不是一个完全打磨好的公开正式版。

当前重点：

- 让安装和更新变得稳定、无感。
- 让业务线在真实用户电脑上更稳定。
- 把普通界面里的技术术语降下来。
- 补齐中文、英文、俄语的表达。
- 让 Claude Code 和 Codex 的行为更可预期。

如果你遇到问题，请在 Issue 中提供复现步骤和日志。

## 如何反馈

具体 Bug 请用 GitHub Issues：

- 安装失败。
- PigBun 无法启动。
- 聊天或业务线行为异常。
- 检查更新找不到最新版本。
- 页面文案让人困惑或没有翻译。

产品想法、使用问题、工作流案例和长反馈，可以使用 GitHub Discussions。

分享日志时，请先移除密钥、token、密码、客户数据和私有仓库地址。
